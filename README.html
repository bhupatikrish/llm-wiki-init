What is this?

Most tools make you search raw documents every time you ask a question. This is different. Claude reads each source once, extracts what matters, and weaves it into a growing wiki of interlinked pages. Every new source makes the wiki richer. Every question you ask can be filed back as a permanent page.

The key idea: Knowledge accumulates instead of being rediscovered from scratch. You supply the sources and questions. Claude handles the summarizing, cross-referencing, filing, and consistency.

This folder is a reusable template. Copy it to create different wikis — AI research, personal notes, competitive analysis. Only the top-level category names need to change between instances.

The raw/ folder — capability tree

Files in raw/ are organized in a business capability hierarchy — up to 4 levels deep. Claude builds this tree dynamically as you ingest. You only provide the top-level names.

raw/ capability tree — example

The wiki/ pages — five types

sources/
One page per ingested document — summary, key points, link back to raw file
concepts/
Technique or idea pages — what it is, how it works, which sources cover it
entities/
Specific models, people, organizations and what the wiki knows about them
analyses/
Comparisons, discoveries, and answered questions filed back from /query
index · log · overview
Catalog, history, and evolving synthesis of everything in the wiki

Every page has a small YAML header with type, tags, and dates. Source pages also include a raw_path — a direct link back to the original file in raw/. Claude uses this when a query needs more detail than the summary provides.

Setup

  1. Copy the template 
    cp -r path/to/init/ my-wiki/
cd my-wiki/
  2. Set your domain — one edit, 3–5 lines

    Open CLAUDE.md, find the Seed Taxonomy section, and replace the top-level categories:

    # Seed Taxonomy — AI/LLM Wiki   ← change the comment
llm                               ← replace these with your
agents                            ← domain's root categories
infrastructure                    ← (3–5 items)

    Other examples: health goals learning for a personal wiki, or product engineering customers for a business wiki. Levels 2–4 grow automatically from your content.

  3. Install Python tools 
    bash tools/setup.sh

    Creates tools/.venv/ with markitdown (converts files/URLs to markdown) and rank_bm25 (search). Takes about 30 seconds.

  4. Open in Claude Code

    Open the wiki folder as the project root. The /ingest-files, /ingest-links, /query, /lint, and /reorg commands become available immediately.

Daily use

Add a source — file

  1. Copy or save a file into staging/files/ (PDF, markdown, Word doc, text)
  2. Run /ingest-files in Claude Code
  3. Claude reads it, decides where it belongs in raw/, moves it there, and writes wiki pages — all in one pass
  4. Browse updated pages in wiki/

Add a source — link

  1. Open staging/links.md and paste a URL on its own line
  2. Run /ingest-links
  3. Claude downloads the page as markdown, then processes it like a file. The URL is removed once done.

Ask a question

/query What is the difference between Graph RAG and standard RAG?

Claude reads relevant wiki pages and answers with links to sources. If the wiki summary isn't detailed enough, it reads the original raw file. If the answer is useful to keep, it's filed as a new page in wiki/analyses/.

Keep the wiki healthy

Run /lint occasionally — Claude scans for orphan pages, broken links, contradictions, stale pages, and concepts that need their own page. Run /reorg when raw/ structure feels off — Claude proposes changes and waits for your approval before moving anything.

Keeping knowledge current

In fast-moving fields like AI/LLM, sources from a few months ago can already be partially outdated. The wiki has three mechanisms to handle this without losing history.

Automatic update check on every ingest

When you ingest a new source, Claude automatically checks existing related pages for conflicts. If the new source updates a claim, it rewrites that section in-place. If it contradicts something, both positions are noted with source dates. If it directly supersedes an older source, the old source page is quietly marked status: superseded with a link to the new one. No extra steps needed.

Page freshness — last_verified

Every concept and entity page carries a last_verified date. When /lint runs, it surfaces pages not reviewed in 60–90 days as candidates for refresh. In AI/LLM, 60 days is a reasonable threshold.

Knowledge lifecycle

Targeted refresh — /refresh <page>

When you know a topic has moved on, run /refresh against the relevant page:

/refresh rag
/refresh GraphRAG

Claude reads the page, finds all contributing sources, scans raw/ for newer files in the same topic area, and rewrites stale claims. The main body always reflects current best understanding. Anything removed from the main body moves to a ## Changelog section — the history is preserved, just not front and centre.

Source pages are never deleted

Source pages (wiki/sources/) are permanent records of what a document said at a point in time. They are never rewritten. When newer work replaces them, they simply gain a superseded_by link in their header — giving you a full audit trail of how the field's thinking evolved.

SituationWhat happens
New source adds detail to a conceptConcept page expanded; sources count incremented
New source updates a specific claimClaim rewritten in-place; Changelog entry added
New source contradicts a claimBoth positions noted with source dates; current understanding prevails in main body
New source entirely supersedes an old oneConcept page rewritten; old source page gets status: superseded + superseded_by link
Technique becomes obsolete (not wrong)## Evolution section added noting when and why it was superseded

How it works

Single-pass ingest

Each source is read exactly once. In that single read, Claude simultaneously decides where the file belongs in raw/ and writes all the wiki pages. No re-reads — the content is already in context.

Ingest — one read, two outputs

Taxonomy Architect — dynamic classification

You never manually organize raw/. Claude classifies each source during ingest using raw/taxonomy.md as a reference. New sub-categories are created when needed. The tree grows from your content.

How a source gets classified

Query with raw fallback

Queries answer from the wiki first — it's fast and covers most questions. When more depth is needed, Claude follows the raw_path in the source page's frontmatter straight to the original file in raw/.
